Plate Mapper

Convert a plate map file to a mapping file.

Usage

Execute this command in terminal:

python plate_mapper.py -i input -t barseq -o output

input, barseq and output are text files in tab-separated values (TSV, also known as tab-delimited) format, in which table fields are separated by the TAB key. Mainstream spreadsheet editors such as Microsoft Excel and Google Sheets can save a spreadsheet as this format.

  • B2:M9, B12:M19...: Plate map region. Each cell contains a sample ID or is left blank. Plates in the same plate map file must be identical in numbers of columns and rows (e.g., 12 x 8).
  • B1:M1, B11:M11...: Column IDs. They must be incremental integers. The script will consider a row as column header if the second cell is "1". The script will count the number of columns until reaching the first cell that is not a number (here N1).
  • A2:A9, A12:A19...: Row IDs. They must be letters in alphabetical order.
  • N2, N12, N23, N34: Primer plate (aka. barcode sequence template plate) ID. Must be an integer. The script will look for this number in rows starting with "A".
    • Can be in any order, and the script will re-order the plates accordingly from low to high.
    • Can omit some numbers, and the corresponding space will be left blank.
  • O2:AA2, O12:AA12...: Values of properties of this plate. They will be appended to the mapping file after each row AS IS.
  • O1:AA1, O11:AA11...: Names of the properties. They are only for the user's reference. The script will not parse them.
  • N3:AA9, N13:AA19...: Doodle region. The user can make any comments in this region.
  • Rows 10, 20-21, 31-32: Optional empty rows. Zero to multiple empty rows are allowed between plates.
  • Formatting will not affect the result. The user is free to use text color, background color, border color and other approaches to mark any cell in the spreadsheet.
